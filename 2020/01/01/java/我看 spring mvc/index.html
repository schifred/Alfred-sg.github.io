我看 spring mvc

发表于 2020-01-01 | 分类于 java |

spring mvc 基于 前端控制器模式 设计，通过 DispatcherServlet 这个前端控制器将请求交给可配置的委托组件处理，这就能支持请求处理的灵活性。DispatcherServlet 中请求映射、视图解析、异常处理等功能所使用的组件都通过 spring 机制发现。

why

@EnableWebMvc

@EnableWebMvc 注解会通过 @Import 加载配置类，配置类又会主动查找添加了 @Controller、@RestController 注解的 bean，并注册到 HandlerMapping 中，以便在请求触达时使用。

DispatcherServlet 初始化

DispatcherServlet 处理请求

  1. 请求触达，DispatcherServlet#doGet 等方法接受到请求。
  2. DispatcherServlet#doDispatch 阶段，路由找出 Handler（即 Controller）。
  3. DispatcherServlet#doDispatch 阶段，使用 HandlerAdapter 作子流程处理。
  4. DispatcherServlet#doDispatch 阶段，使用 Handler 作主流程处理。
  5. DispatcherServlet#doDispatch 阶段，视图渲染、处理响应。
  • HandlerMapping：请求到一组拦截器的映射，拦截器包含对请求的前置、后置处理。
  • HandlerAdapter：辅助 Handler，用于处理如 Controller 上注释解析等子流程，使 Handler 专注于处理请求的主流程。
  • HandlerExceptionResolver：错误处理，将他们转交给 Handler、视图等。
  • ViewResolver 视图解析器；LocaleResolver, LocaleContextResolver 国际化；ThemeResolver 个性化；MultipartResolver 解析文件上传请求；FlashMapManager 闪存。
