umi-plugin-qiankun

发表于 2020-02-20 | 分类于 前端 |

umi-plugin-qiankun 基于 qiankun，允许在 umi 环境中创建微服务。

示例

按官方示例，搭建 umi 微服务可基于以下流程实现：

  1. 主应用 .umirc.js 添加 @umijs/plugin-qiankun 插件配置 master 内容：注册子应用（应用名、html 地址、路由前缀、路由方式、挂载节点、自定义属性等）、启动沙箱、启动预加载、启动异步渲染等。（备注：子应用也可以在运行时动态注册）
  2. 子应用 .umirc.js 添加 @umijs/plugin-qiankun 插件配置 slave 内容：keepOriginalRoutes。
  3. 子应用入口文件中导出 qiankun = { bootstrap, mount, unmount }，非必要。

源码

基于 qiankun、umi 的运作机制，使用 umi-plugin-qiankun 搭建微服务就是在运行时从 .umirc.js 获取配置项，然后调用 qiankun 的接口注册子应用并启动微服务。想要在运行时从 .umirc.js 获取配置项，umi 提供的实现方式是利用 api.writeTmpFile 接口将配置项转化成 json 临时文件，然后在运行时插件中加载该临时文件。运行时配置则可以通过

umi-plugin-qiankun 会为 umi 上下文注入 qiankun-master 主应用（非运行时）插件、qiankun-slave 子应用（非运行时）插件。当配置项变更时，umi-plugin-qiankun 会利用 api.onOptionChange、api.changePluginOption 的特性，主动更新 qiankun-master、qiankun-slave 插件所使用的配置数据（即读取 .umirc.js 配置文件中有关 @umijs/plugin-qiankun 插件的 master、slave 配置）。qiankun-master、qiankun-slave 两者都会通过 api.addRuntimePlugin 接口注册运行时插件，主应用中添加运行时插件的目的在于使用 .umirc.js 配置注册子应用；子应用添加运行时插件的目的在于在多个应用中共享数据。
